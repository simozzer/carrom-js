Carrom-JS · Chaos Map

Every pixel is one shot fired into a rack of coins. Left→right = aim angle, bottom→top = power. Colour = which coins it pockets (incl. the striker — a scratch); brightness = number of collisions. The frilly, fractal boundaries are sensitive dependence: a hair's change in aim flips the whole outcome.

Dark blue regions = no coin potted (still textured by how many times things bounced). Desaturated/grey bands = the striker scratched. This is dissipative billiards (friction + restitution < 1), so it's transient chaos — chaotic until everything comes to rest. The engine predicts every collision analytically, so the structure you see is real, not solver noise. · back to the game
